# Internationalization Guide

MilkTea Premium includes a scalable multilingual architecture.

## Default Language

Default language:

```text
vi
```

Supported languages are registered in:

```text
src/locales/index.js
```

## Locale Files

Current locale files:

```text
src/locales/vi.js
src/locales/en.js
```

Each locale exports an object with matching nested keys.

## Translation Function

Use:

```js
import { t } from '../utils/i18n.js';

t('navbar.menu')
t('buttons.addToCart')
t('products.reviewCount', { count: 24 })
```

If a key is missing, `t()` returns the key path. This makes missing translations easy to identify during development.

## Language Store

Language preference is managed by:

```text
src/store/languageStore.js
```

Responsibilities:

- read current language
- persist language preference
- notify subscribers
- update `document.documentElement.lang`
- update document title and description

## Language Switcher

The reusable language switcher lives in:

```text
src/components/LanguageSwitcher.js
```

It is rendered in:

- desktop navbar
- mobile drawer

Language changes happen instantly without page reload.

## Adding a Language

1. Create a new locale file:

```text
src/locales/fr.js
```

2. Export a locale object matching existing keys:

```js
export const fr = {
  navbar: {
    home: 'Accueil'
  }
};
```

3. Register it in `src/locales/index.js`:

```js
import { fr } from './fr.js';

export const LANGUAGES = [
  { code: 'vi', labelKey: 'common.vietnamese' },
  { code: 'en', labelKey: 'common.english' },
  { code: 'fr', labelKey: 'common.french' }
];

export const locales = {
  vi,
  en,
  fr
};
```

4. Add the language label key to all locale files.

## Translation Rules

- Put fixed UI labels in locale files.
- Put editable marketing/business content in `siteConfig`.
- Use natural, human-written copy.
- Keep key groups organized by feature, for example `navbar.*`, `buttons.*`, `menu.*`, `checkout.*`.
- Avoid concatenating translated sentence fragments when possible.
